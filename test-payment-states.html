How Payment Status Validation Works

Status Detection
  • URL Parameters: Checks ?status=cancelled or ?status=failed
  • Database Validation: Verifies transaction status in database
  • Vote Verification: Confirms votes were actually created
  • Fallback Handling: Handles edge cases and errors gracefully
User Experience
  • Success: Shows payment summary and vote details
  • Cancelled: Clear message with retry option
  • Failed: Error explanation with support info
  • Pending: Processing message for delayed payments
